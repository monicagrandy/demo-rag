### Real Estate Negotiation Problem

- Building simulator: Two AI agents negotiate house price
  - Property: 7742 Evergreen Terrace, 4BR/3BA, 2400 sq ft
  - Listed at $485,000
- Buyer agent constraints:
  - Opens at $425,000 (10-12% below asking)
  - Hard ceiling: $460,000 (walks away above this)
- Seller agent constraints:
  - Counters at $477,000 (slight concession from list)
  - Floor price: $445,000 (walks away below this)
- Zone of Possible Agreement (ZOPA): $15,000 band between $445K-$460K

### Why Naive Agents Fail

- Gap between demo and production rarely model quality (only ~2% of Cloud Code is model)
- Real challenges in production systems:
  1. Control flow: Guaranteed loop exits required
  2. Parsing: Avoiding brittle regex on unstructured text
  3. Termination: Clean conversation endings vs infinite loops
- Common failure patterns:
  - Raw string parsing with brittle regex
  - No schema validation on LLM responses
  - Hard-coded prices instead of live API data
  - Missing observability and evaluation frameworks

### Model Context Protocol (MCP) Fundamentals

- Standard protocol for agent tool access without knowing data source
- Solves M×N integration problem (5 agents × 10 APIs = 50 integrations)
- Three core operations:
  1. Initialize: Handshake with capability negotiation
  2. List: Server returns typed JSON schema for available tools
  3. Call: Client invokes tools with validated arguments

### MCP Primitives and Content Types

- Four primitive types:
  1. Tools: Actions with side effects (phone calls, API calls)
  2. Resources: Static documents for LLM context (background info)
  3. Prompts: Behavioral instructions based on conditions
  4. Sampling: Server requests LLM processing (rarely used)
- Three content types: Text, Image (base64), Embedded resources
- Three transport modes: stdio (local), SSE (legacy HTTP), streamable HTTP (recommended)

### Information Asymmetry Implementation

- Buyer agent: Access only to pricing server tools
  - get_market_price for justifying offers
  - calculate_discount for offer ranges
- Seller agent: Access to both pricing and inventory servers
  - Additional access to get_minimum_acceptable_price
  - Inventory data should remain hidden from buyer
- Enforced through Google ADK before_tool_callback functionality
