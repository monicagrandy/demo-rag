### System Architecture

- Router agent as first point of contact
  - Classifies user requests into one of three categories
  - Routes to appropriate specialized agent
  - No supervision or output validation
- Flow: User Input → Router → Selected Agent → Final Output
  - No return to router after agent completes task
- State management tracks:
  - User input (original request)
  - Route decision
  - Final output
  - Messages list for conversation history

### Implementation Walkthrough

- Template notebook provided to guide step-by-step development
- Start with state definition, then router, then individual agents
- Router implementation:
  - Simple classification prompt
  - Returns one word: “SEO blog writer”, “x blog writer”, or “general”
  - No historical context needed for routing decisions
- General handler:
  - Access to full conversation history
  - Brief, contextual responses
  - Mentions specialization in content writing if relevant

### Tools and Technical Setup

- Required packages: LangGraph, LangChain, LangChain-OpenAI, Deep-Agents
- API keys: OpenAI (GPT-4 recommended for testing, GPT-4o for final runs)
- Search capabilities via Tavily API
  - Internet search tool for direct web results
  - Deep research agent for comprehensive analysis
- SEO Blog Writer gets both tools:
  - Research tool for content insights
  - Internet search for SEO keywords and trending terms

### Key Technical Considerations

- History management critical for cost and performance
  - Consider token limits and summarization after N turns
  - Long conversations can disconnect from system instructions
- Temperature settings:
  - Lower for research agents (factual, deterministic)
  - Standard for creative content generation
- Tool calling loops handle multi-step processes automatically
- Deep agents provide extensive research vs single LLM calls
