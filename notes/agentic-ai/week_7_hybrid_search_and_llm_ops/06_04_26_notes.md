### Hybrid Search Architecture Overview

- RAG system components:
  - Large language model as “brain”
  - Vector databases as external knowledge tools
  - Search as critical component for context retrieval
- Why hybrid search matters:
  - Dense search: semantic similarity matching
  - Sparse search: exact keyword/term matching
  - Combined approach captures different search needs
- Production considerations: search quality directly impacts RAG system performance

### Assignment Context & Dataset

- Medical dataset from PubMed (vs previous Amazon fans dataset)
- Data structure:
  - Context column contains dictionary with medical information
  - Mesh terms (standardized medical terminology)
  - Questions with yes/no/maybe answers
  - Long-form answers based on context
- Using 100 documents for indexing exercise
- Focus on context + mesh terms + long answers for search corpus

### Technical Implementation Details

- FastEmbed library for CPU-based model inference
  - Dense vectors: BGE model (1024 dimensions)
  - Sparse vectors: SPLADE model for query expansion
- Qdrant vector database for storage:
  - Supports both dense and sparse indexes
  - Cosine similarity for dense search
  - In-memory processing for assignment
- Two-stage process: offline indexing → online search

### Search Process & Query Expansion

- SPLADE functionality:
  - Expands query terms with semantically related words
  - Example: “capital of France” → includes “city”, “Eiffel Tower”
  - Uses transformer model for contextual expansion
- Misspelling handling:
  - BERT-based models consider full sentence context
  - Some resilience through contextual understanding
  - Production systems should include spell-check layer

### Reciprocal Rank Fusion (RRF) Algorithm

- Combines two ranked result lists with different scoring scales
- Formula: 1/(rank_dense + 60) + 1/(rank_sparse + 60)
- Key characteristics:
  - Uses ranking position, not raw scores
  - Smoothing factor (60) prevents first-rank dominance
  - Favors results appearing in both searches
- Limitations:
  - No weighting between search types
  - Discards score magnitude information
  - 15+ year old algorithm, newer ML approaches available

### Production Scaling Considerations

- Database scaling challenges:

  - Cost optimization strategies

- Search-specific evaluation:
