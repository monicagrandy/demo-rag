### Voice Agent Architecture Fundamentals

- Two main approaches for voice agents:
  - Separate models for speech-to-text → LLM → text-to-speech
  - Higher control and transparency
  - Better observability for enterprise scaling
  - Adds latency but allows model swapping

### LangGraph Sub-graphs & Communication Patterns

- Two implementation patterns:
  - Different state schemas between parent/sub-graph
  - Requires wrapper functions for state conversion
- Conditional edges used when target node determined dynamically at runtime
- Static edges for predetermined workflow paths

### Human-in-the-Loop & Interrupts

- Requires stateful agents with memory/persistence support
- Interrupt workflow:
  - Node calls interrupt() method
  - Graph execution pauses, state saved with thread ID
  - Returns context to caller explaining pause reason
  - Resume using same thread ID with user input
- Node restarts from beginning when resumed
- Critical for high-stakes actions (purchases, deployments, emails)

### E-commerce Assistant Architecture

- Multi-agent system with 4 main nodes:
  1. Orchestrator - LLM classifier routing queries to appropriate agents
  2. Product Agent - RAG-powered product search (Chroma vector DB)
  3. Support Agent - Order status + human escalation tools
  4. Synthesizer - Combines multi-agent responses
- Supports parallel agent execution when both required
- Uses OpenAI models: GPT-4.0, Whisper, TTS, text-embedding-3-small

### Code Structure & Implementation

- State management through AxiomCardState class
